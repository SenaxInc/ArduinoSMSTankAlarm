## Summary

v1.1.6 is a **major security and hardening release** driven by a triple AI code review (Gemini 3.1 Pro, GPT-5.3-Codex, Claude Opus 4.6) on March 12, 2026. It resolves all CRITICAL and HIGH security findings — eliminating PIN storage in the browser, removing session tokens from URLs, enforcing single-session login with server-generated tokens, adding relay command target verification, and requiring PIN enrollment on fresh install. Additionally, it delivers DFU error auto-recovery, tank registry deduplication, per-monitor stuck-sensor detection, Viewer metadata enrichment, deterministic MAC generation, and notefile schema versioning.

No breaking changes to configuration files. Browser sessions will be invalidated after upgrade (users must re-login).

---

## 🔒 CRITICAL Security Fixes

### S-1: Remove PIN from Browser localStorage

- `localStorage.setItem("tankalarm_token", pin)` replaced with a `"1"` session flag
- All `pin: token || null` references in fetch calls replaced with `pin: null`
- Cancel/refresh API calls now prompt for PIN per-action instead of using a cached credential
- **Impact:** Raw 4-digit PIN no longer persists in browser storage

### S-2: Remove Session Token from URL Query Strings

- `handleSessionCheck()` now reads session from `X-Session` HTTP header instead of URL `?token=` parameter
- All 11 JavaScript `fetch('/api/session/check?token='+sid)` calls changed to header-based authentication
- `readHttpRequest()` extended to parse and pass through the `X-Session` header
- **Impact:** Session tokens no longer appear in browser history, server logs, or Referer headers

### C-1: Relay Command Target-UID Verification (Client)

- Added `doc["target"]` UID check at the top of `processRelayCommand()`
- Compares against `gDeviceUID`; drops and logs mismatched commands
- **Impact:** Prevents route misconfiguration from actuating relays on unintended clients

## 🔐 HIGH Security Fixes

### S-3: Require PIN Enrollment on Fresh Install

- Fresh-install login now requires a valid 4-digit PIN
- Returns HTTP 400 "Please set a 4-digit PIN on first login" instead of granting a session with a blank PIN
- **Impact:** Eliminates unauthenticated access on new deployments

### Single-Session Login with Server-Generated Tokens

- Introduced 16-character server-side session token generated from `micros()`, `millis()`, and ADC entropy
- Constant-time comparison for session validation (prevents timing attacks)
- `requireValidPin()` rejects requests when no active session exists (returns 401)
- Logout POSTs to `/api/logout` and clears both server token and client session
- Client JavaScript polls `/api/session/check` every 30 seconds; revalidates on tab visibility change to expire stale sessions
- **Impact:** Only one browser session active at a time; stale tabs auto-expire

### Session Token Entropy Upgrade

- `generateSessionToken()` upgraded to 64-bit LCG with ADC entropy from `VIN_ANALOG_PIN` (A0)
- Configurable entropy pins with improved mixing algorithm
- **Impact:** Stronger session tokens reduce brute-force risk

### X-Session Header on All Embedded Pages

- Wrapped `window.fetch` in `SERVER_SETTINGS_HTML` and `EMAIL_FORMAT_HTML` to auto-attach `X-Session` header from localStorage
- 401 responses clear local storage and redirect to login page
- **Impact:** Consistent session enforcement across all web UI pages

### PIN Auth for Debug/Tanks Endpoint

- Changed `/api/debug/tanks` from GET+query-string PIN to POST-only with PIN in JSON body
- **Impact:** Eliminates PIN exposure in URLs, browser history, and server logs

## 🔧 HIGH Bug Fixes

### S-4: Config Dispatch/Cache Buffer Size Mismatch (Server)

- `cacheClientConfigFromBuffer()` changed from `void` to `bool` return
- `dispatchClientConfig()` checks return value and responds `PayloadTooLarge` if cache fails
- `DeserializeJson` failure now returns `false` instead of silently ignoring
- **Impact:** Configs exceeding cache buffer are properly rejected instead of silently written as 0-byte files

### S-5: Config Retry Cadence Documentation Fix (Server)

- HTTP response text corrected from "auto-retry every 60s" to "auto-retry every 60 min" to match actual `3600000UL` retry interval

### Client Config Buffer Increase

- Added `EXTENDED_FIELDS_MAX` constant and increased `lineBuffer` size in `loadClientConfigSnapshots()`
- Per-entry buffer increased from ~680 to ~1700 bytes to accommodate `payload` field (1536 bytes max)
- **Impact:** Prevents silent truncation of extended fields (`pd`, `configVersion`, `epoch`, `ackStatus`, `dispatchAttempts`, `dispatchEpoch`)

## 🚀 New Features

### DFU Error Auto-Recovery (Server)

- Added `gDfuError` to capture last DFU error string from Notecard `dfu.status` response
- Displays `dfuError` in `/api/dfu/status` JSON response and web UI for diagnostics
- When a manual DFU check is requested and the Notecard is in an error state:
  1. Sends `dfu.status {"stop": true}` to abort/clear error
  2. Toggles `card.dfu` off then on to re-enable outboard DFU
  3. Triggers `hub.sync` for fresh firmware catalog from Notehub
  4. Resets all local DFU state variables for fresh discovery
- **Impact:** DFU errors no longer require manual serial intervention to recover

### Tank Registry Deduplication (Server)

- `deduplicateTankRecordsLinear()` removes duplicate `clientUid + tankNumber` entries and rebuilds the hash table at boot
- Merges duplicates on load, keeping the record with the latest `lastUpdateEpoch`; marks registry dirty for re-save
- Added `/api/debug/tanks` POST endpoint (`{"action":"dedup"}`) to force deduplication on demand
- **Impact:** Eliminates ghost records from corrupted registry files or Notecard backlogs

### Per-Monitor Stuck Sensor Detection (Client + Server)

- Added `stuckDetection` boolean flag to `MonitorConfig`, enabled by default
- Checkbox in server sensor configuration UI to enable/disable per monitor
- Persisted and loaded in config JSON
- `validateSensorReading()` respects the setting before counting identical readings
- Measurement unit (`mu`) included in unload events
- **Impact:** Stuck detection can be selectively disabled for monitors where constant readings are expected (e.g., overflow tanks)

### Notefile Schema Versioning (Common Library)

- Added `NOTEFILE_SCHEMA_VERSION` (= 1) to `TankAlarm_Common.h`
- Client's `publishNote()` injects `_sv` into every outbound note body
- Server's 9 outbound `note.add` call sites stamp `_sv` via `stampSchemaVersion(body)` before `JAddItemToObject`
- **Impact:** Enables future backward-compatible payload format changes

### Boot-Time `.tmp` Orphan Cleanup (Common Library)

- Added `tankalarm_posix_cleanup_tmp_files()` and `tankalarm_littlefs_cleanup_tmp_files()` to `TankAlarm_Platform.h`
- Server `setup()` calls cleanup for all 9 known config file paths after `initializeStorage()`
- If original file is intact, orphan `.tmp` is removed; if only `.tmp` exists, rename is completed
- **Impact:** Failed atomic writes no longer leak flash storage

## 📺 Viewer Improvements

### Metadata Enrichment (V-2 / V-3)

- `TankRecord` struct expanded with `objectType[16]`, `sensorType[16]`, `measurementUnit[16]`, `change24h`, `hasChange24h`
- `handleViewerSummary()` reads `ot`/`st`/`mu`/`d` fields from Notecard payload
- `sendTankJson()` serializes new fields conditionally
- PROGMEM JavaScript `format24hChange()` replaces hardcoded `--` with signed delta display

### JsonDocument OOM Fix (V-4)

- Changed `new JsonDocument()` to `new (std::nothrow) JsonDocument()` so the null check actually works on out-of-memory
- Added `#include <new>`

### Deterministic MAC from Notecard UID (V-6)

- Added `deriveMacFromUid()` — hashes `gViewerUid` via DJB2 into MAC bytes 2–5
- Byte 0 = `0x02` (locally administered), eliminating hardcoded MAC collision risk across multiple Viewer devices
- Called in `setup()` after `initializeNotecard()`, before `initializeEthernet()`

### Ethernet Init Retry (V-5)

- `initializeEthernet()` now retries up to 3 times with increasing delays (5s, 10s, 15s) on failure

## 🔧 Client Reliability Improvements

### Relay Command Cooldown (C-4)

- Added `RELAY_COMMAND_COOLDOWN_MS` (5000 ms) to `TankAlarm_Config.h`
- `processRelayCommand()` tracks last execution time; commands within cooldown window are rejected and logged
- **Impact:** Prevents rapid toggling from stale queued Notes

### I2C Sensor Recovery Max Attempts (C-3)

- Added `I2C_SENSOR_RECOVERY_MAX_ATTEMPTS` (5) constant to `TankAlarm_Config.h`
- Sensor-only recovery tracks total attempts; after 5, logs "permanent fault" and stops retrying
- Counter resets when sensors fully recover
- **Impact:** Prevents infinite recovery loops from permanently faulted sensors

### Solar Sunset State Persistence (C-7)

- `saveSolarStateToFlash()` now persists the `sunsetActive` flag
- Initial sunset detection triggers an immediate save so reboots during the confirmation window resume correctly
- **Impact:** Power cycles during sunset detection no longer lose state
