# Release Notes — v1.1.5

**Date:** March 10, 2026  
**Components:** Client, Server, and Viewer Firmware (`TankAlarm-112025-*-BluesOpta`)  
**Status:** Stable Release  
**Previous Release:** v1.1.4 (March 1, 2026)

---

## Summary

v1.1.5 is a **telemetry enrichment and operations release** that adds object type and measurement unit metadata to all telemetry and alarm payloads, introduces automatic pruning of orphaned tank records on config dispatch, documents external antenna installation, and re-enables auto-DFU in the client main loop.

No breaking changes. All existing configurations are fully backward-compatible.

---

## 🚀 New Features

### Measurement Unit & Object Type in Telemetry (Client + Server)

Client telemetry and alarm payloads now include object type (`ot`) and measurement unit (`mu`) fields, enabling downstream consumers (Viewer, Notehub routes, dashboards) to display data with proper units and context without needing to look up the client config.

- **Client:** Parses and prefers `measurementUnit` from config with fallback derivation from `objectType` and `sensorRangeUnit`; accepts legacy `"monitorType"` and `"rpm"` aliases for backward compatibility
- **Client:** Includes `ot` (object type) and `mu` (measurement unit) in every telemetry note and alarm payload
- **Server:** Accepts both `"number"` and `"tankNumber"` fields from legacy client configs
- **Server:** Derives `objectType` and `measurementUnit` from cached client config snapshot when telemetry omits them (backward compatibility for pre-1.1.5 clients)

### Orphaned Tank Record Pruning (Server)

- Added `pruneOrphanedTankRecords()` — automatically removes stale `TankRecord` entries when a client config is dispatched or ACKed
- Prevents accumulation of ghost records from sensor reconfiguration (e.g., reducing from 4 monitors to 2)
- Triggers on config dispatch and config ACK receipt

### Auto-DFU Re-enabled (Client)

- Re-enabled automatic DFU application in the client main loop
- Applies Notecard/Notehub firmware updates automatically when `gDfuUpdateAvailable` is set
- Guarded by power state (skipped in CRITICAL), Notecard availability, and DFU-in-progress checks
- Previously disabled during v1.1.3–v1.1.4 development for stability testing

## 📦 Upgrade Instructions

1. **Flash all three firmware images** (Client, Server, Viewer) to pick up the version bump.
2. **No config reset required** — existing configs are fully backward-compatible.
3. Clear browser cache or hard-refresh (Ctrl+F5) on the Server web UI.
